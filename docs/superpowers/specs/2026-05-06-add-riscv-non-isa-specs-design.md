# Add RISC-V non-ISA specs to Docusaurus site

## Goal

Add four RISC-V non-ISA specification repositories as first-class documentation roots with their own sidebars and navbar entries:

- `riscv-non-isa/riscv-asm-manual` → `/docs/asm-manual`, label `Assembly`
- `riscv-non-isa/riscv-sbi-doc` → `/docs/sbi`, label `SBI`
- `riscv-non-isa/riscv-iommu` → `/docs/iommu`, label `IOMMU`
- `riscv-non-isa/riscv-trace-spec` → `/docs/trace`, label `Trace`

Current ISA roots remain unchanged:

- `/docs/unprivileged`
- `/docs/privileged`

## Approach

Use the existing single Docusaurus docs plugin. Generate each spec into its own `docs/<slug>/` directory with a generated `sidebar.json`, then import those sidebars in `sidebars.js` and expose them via `docSidebar` navbar items.

This keeps all documentation searchable under `/docs`, avoids multi-plugin search/config complexity, and matches the current unprivileged/privileged pipeline.

## Generation design

Extend `gen-mdx.sh` into a small manifest-style generator while preserving existing behavior.

Source directories default to sibling checkouts and can be overridden with environment variables:

| Env var | Default |
|---|---|
| `MANUAL_DIR` | `../riscv-isa-manual` |
| `ASM_MANUAL_DIR` | `../riscv-asm-manual` |
| `SBI_DOC_DIR` | `../riscv-sbi-doc` |
| `IOMMU_DIR` | `../riscv-iommu` |
| `TRACE_SPEC_DIR` | `../riscv-trace-spec` |

Known source roots:

| Slug | Root AsciiDoc | Extra handling |
|---|---|---|
| `asm-manual` | `src/prelude.adoc` | `asciidoctor-lists` |
| `sbi` | `riscv-sbi.adoc` | create `autogenerated/revision.adoc-snippet`; use `src/references.bib` |
| `iommu` | `src/riscv-iommu.adoc` | `asciidoctor-bibtex`, `asciidoctor-lists` |
| `trace` | `header.adoc` | no bibliography |

Each spec gets an independent image namespace:

- `/img/riscv-asm-manual/`
- `/img/riscv-sbi-doc/`
- `/img/riscv-iommu/`
- `/img/riscv-trace-spec/`

Kroki diagram cache remains global for the site under the existing Docusaurus `remark-kroki-plugin` configuration. Upstream image files use per-spec namespaces; generated diagram SVG cache can stay shared.

## Docusaurus design

- Import generated sidebars for all six roots in `sidebars.js`.
- Keep the existing `withPrefix()` behavior for generated sidebar links and clickable category headers.
- Add navbar entries for Assembly, SBI, IOMMU, and Trace after existing Unprivileged/Privileged entries.
- Keep search configured for `docs` route.

## CI design

Update the build workflow to treat all five upstream repos as inputs:

1. Query HEAD SHA for each upstream repository.
2. On scheduled runs, compare deployed stamp to combined SHA stamp.
3. Checkout all upstream repos when rebuild is needed.
4. Pass source directory env vars to `gen-mdx.sh`.
5. Write combined upstream stamp to `static/built-upstream-shas.txt`; keep `static/built-manual-sha.txt` only if backward compatibility is needed.

Manual and push builds still always rebuild.
