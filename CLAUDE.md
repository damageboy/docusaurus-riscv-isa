## What This Project Is

A Docusaurus 3.x site that renders the [RISC-V ISA Manual](https://github.com/riscv/riscv-isa-manual) as a searchable, navigable web site. The manual sources are AsciiDoc; a custom `asciidoctor-mdx` tool converts them to MDX files consumed by Docusaurus. The generated docs live in `docs/unprivileged/` and `docs/privileged/` (gitignored — never hand-edit them).

## Commands

```bash
bun run start       # dev server with hot reload
bun run build       # production build → build/
bun run serve       # serve the production build locally
bun run clear       # clear Docusaurus cache
```

### Regenerating docs from source

The `gen-mdx.sh` script converts the upstream AsciiDoc manual into MDX. It requires:
- The `riscv-isa-manual` repo checked out at `../riscv-isa-manual` (sibling directory)
- `asciidoctor-mdx` available at `~/projects/asciidoctor/wrappers/asciidoctor-mdx` (overridable via `ASCIIDOCTOR_MDX` env var)

```bash
./gen-mdx.sh        # regenerate docs/ and static/img/riscv-isa/
```

**After every change to any file in this repo, you must run `bun run build` before declaring the task complete.** No exceptions. A task is not done until the build passes.

If the change also affects doc generation (edits to `gen-mdx.sh`), run the full pipeline:

```bash
./gen-mdx.sh && bun run build
```

## Architecture

### Content pipeline

```
riscv-isa-manual/src/{unpriv,priv/priv}.adoc
    → gen-mdx.sh (wraps with symbols.adoc, calls asciidoctor-mdx)
    → riscv-isa-manual/build/{unpriv,priv}/*.mdx + sidebar.json
    → docs/{unprivileged,privileged}/*.mdx + sidebar.json
    → Docusaurus build
```

`gen-mdx.sh` also copies images to `static/img/riscv-isa/` and diagram render cache goes to `static/img/riscv-isa/kroki/` (via `remark-kroki-plugin`).

### Sidebars

`sidebars.js` imports `docs/unprivileged/sidebar.json` and `docs/privileged/sidebar.json` (generated by `asciidoctor-mdx`) and processes them with `withPrefix()`. That function:
1. Prepends `/docs` to all `href` fields.
2. Converts category items whose `link.type === 'link'` (not valid in Docusaurus schema) into `customProps.href`, removing the `link` field so Docusaurus validation passes.

### Swizzled component

`src/theme/DocSidebarItem/Category/index.js` — wraps the original Docusaurus `DocSidebarItemCategory` to read `customProps.href` and inject it as `item.href`, making category headers in the sidebar clickable links (pointing to the section anchor page) rather than non-clickable collapse toggles.

### Remark/Rehype plugins (docusaurus.config.js)

| Plugin | Purpose |
|---|---|
| `remark-math` + `rehype-katex` | LaTeX math rendering |
| `remark-kroki-plugin` | Diagram code blocks (wavedrom, graphviz, bytefield, etc.) rendered via kroki.io, cached as SVGs |
| `@adobe/remark-gridtables` | Pipe-table extension for complex grid tables from AsciiDoc |

Kroki diagrams are identified by language alias (e.g. ` ```wavedrom `) and cached in `static/img/riscv-isa/kroki/`.

### Key config notes

- `future.v4: true` and `future.faster: true` are enabled — Docusaurus v4 compatibility mode.
- `markdown.mdx1Compat.headingIds: true` — required for `{#id}` heading syntax used by the generated MDX.
- `onBrokenLinks: 'warn'` — broken links do not fail the build.
- Blog is disabled.
- Local search is provided by `@easyops-cn/docusaurus-search-local`.
